# Histology AI Service

Сервис автоматической классификации гистологических изображений печени по типам дистрофии на основе ResNet50.

Проект представляет собой полноценный ML backend-сервис с асинхронной обработкой задач, очередями, API, системой пользователей и балансом.

---

# Возможности

- Загрузка whole-slide изображений
- Предобработка гистологических слайдов
- Нарезание гистологического слайда
- AI-классификация изображений
- Асинхронная обработка через Celery
- Система пользователей
- JWT-аутентификация
- Баланс пользователя
- Промокоды
- Docker-инфраструктура
- Мониторинг задач через Flower

---

# ML Pipeline

## 1. Загрузка изображения

Пользователь загружает гистологический слайд (.svs)
---

## 2. Предобработка

Preprocess worker:

- открывает whole-slide изображение через OpenSlide
- разбивает изображение на тайлы 512x512
- сохраняет тайлы во временное хранилище

---

## 3. Инференс

Inference worker:

- загружает тайлы
- выполняет предсказание через PyTorch модель
- агрегирует результаты
- сохраняет итоговый диагноз

---

# Поддерживаемые классы

Модель умеет классифицировать:

- мелкокапельная жировая дистрофия
- крупнокапельная жировая дистрофия
- норма

# Регистрация или вход

После регистрации пользователь получает:

- стартовый баланс (1000 дисфов)
- возможность выполнять предсказания (-100 с баланса за каждое предсказание)
- JWT токен для авторизации

Учитывайте, что при нулевом балансе невозможно сделать предсказание.

# Промокоды

В системе реализована поддержка промокодов. Промокоды позволяют пополнять баланс на 100 или 200 дисфов.

# Celery Workers
1) Preprocess Worker

Отвечает за:

- открытие whole-slide изображений
- генерацию тайлов
- подготовку данных для модели

2) Inference Worker

Отвечает за:

- запуск AI модели
- классификацию тайлов
- подсчёт confidence score
- сохранение результата

Проект полностью контейнезирован в Docker

Документация API: http://localhost:8000/docs (работает при запуске Docker)
Сам сервис Streamlit: http://localhost:8501/ (работает при запуске Docker)

# Взаимодействие с пользователем:

- Пользователь регистрируется
- Загружает слайд (есть ограничение на загрузку: не более 200 МБ)
- Celery preprocess worker создаёт тайлы
- Inference worker запускает модель
- Пользователь получает диагноз
